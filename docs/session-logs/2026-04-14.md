### Contexto

ContinuaciГіn de la migraciГіn completa Supabase вҶ’ DigitalOcean (13/04). Objetivo de esta sesiГіn: diseГұar el sistema multi-rol (admin/teacher/student) con relaciГіn profesorвҶ”estudiante, y preparar entorno de desarrollo local para prototipar la migraciГіn DB antes de tocar producciГіn.

### Decisiones de producto

- **Registro restringido a `@unbosque.edu.co`** вҖ” dominio institucional obligatorio a nivel DB (CHECK constraint) y app.
- **Roles independientes** вҖ” admin y teacher son roles distintos. Un usuario puede tener ambos (Carlos = admin+teacher).
- **Multi-rol por usuario** вҖ” tabla pivote `user_roles` (M:N) reemplaza columna `profiles.role`.
- **RelaciГіn N:M profesorвҶ”estudiante** вҖ” tabla `teacher_students` en vez de `profiles.teacher_id`. Soporta cohort/talleres futuros.
- **Visibilidad**: todos ven la galerГӯa; solo el profesor asignado (o el admin) puede gestionar modelos de un estudiante.
- **Flujo de creaciГіn**:
  - Estudiantes: autoregistro con correo `@unbosque.edu.co`.
  - Docentes: creados por admin.
